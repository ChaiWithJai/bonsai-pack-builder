# Means, Motivation, Opportunity

## Means

We can build this now because the previous demo already produced the hard-won
pattern:

- synthetic-only workflow examples;
- pack-shaped policy, schema, sink, and eval files;
- verifier gate outside the model;
- phone HTTPS observability;
- latency expectation-setting;
- claim guardrails for public demos;
- GitOps discipline around proof before publish.

Deep Agents contributes the harness primitives: planning, virtual filesystem,
subagents, HITL, permissions, memory, and sandbox execution.

SvelteKit contributes an approachable UI layer with addressable routes, progressive
enhancement, and TypeScript.

PrismML's Bonsai-demo repo contributes the ecosystem target: benchmark reports the
community can compare and improve.

## Motivation

The adopter does not want an agent demo. They want:

- a credible healthcare hackathon starter;
- a way to explain first-party edge AI to leadership;
- a pack they can edit without becoming a model/runtime expert;
- proof that sensitive text is handled inside a boundary;
- copy that does not overclaim.

The builder wants:

- a clean harness to extend;
- a place to contribute packs;
- benchmark evidence for Bonsai in a real workflow;
- a path from runtime improvement to visible user value.

## Opportunity

The ecosystem opening is a pack and evidence loop. A user starts with a workflow,
turns it into a pack, runs an eval, collects proof, writes the story, and prepares
a contribution.

Nomad Pack made operational workloads shareable. Packer made machine images
reproducible. Bonsai Pack Builder can make regulated AI workflow boundaries
shareable and measurable.

This demonstrates Prism's founders' thesis in adoption terms: useful intelligence
per unit hardware, power, and workflow friction. The first win is not "Bonsai beats
frontier cloud." The win is "a small model inside a correct harness lets a sensitive
workflow stay closer to the data."
